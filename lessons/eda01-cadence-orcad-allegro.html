Cadence OrCAD + Allegro 设计流程

Lesson EDA01 · EDA 工具实战 · 预计用时 45 分钟

本节目标 掌握 Cadence OrCAD Capture（原理图）和 Allegro PCB Editor（布局）的完整设计流程，能独立完成从原理图到 PCB 投板的操作——重点覆盖模拟电路的常用功能和技巧。

0. Cadence PCB 设计流程总览

Cadence 的 PCB 设计工具链由多个工具组成，核心流程如下：

  OrCAD Capture          Allegro PCB Editor          制造输出
  （原理图设计）                （PCB 布局布线）                （生产文件）
        |                            |                            |
  画原理图 → 生成网表 →  导入网表 → 布局 → 布线 →  出 Gerber → 投板
        |                            |                            |
  创建设备符号                 约束规则设置                 钻孔文件
  分配 PCB 封装                叠层设计                    装配图
  DRC 检查                     DRC 检查                     IPC-356 网表

关键名词解释：

名词全称含义
Netlist（网表）—描述元器件引脚之间连接关系的文件。OrCAD 生成网表，Allegro 读取网表
Footprint（封装）—元器件的 PCB 物理焊盘图形，.dra 文件
Symbol（符号）—原理图中元器件的逻辑图形，.olb 文件
Cross-probe—OrCAD 和 Allegro 之间的交互定位：在原理图中选中一个器件，PCB 中自动高亮
Back annotation—从 PCB 回传信息到原理图（如修改后的位号、交换的引脚）
Constraint约束走线的电气和物理规则（线宽、间距、等长、阻抗等）

1. OrCAD Capture 原理图设计

1.1 界面速览

打开 OrCAD Capture CIS 后，你会看到几个主要区域：

  • 项目管理器（Project Manager）：左侧树形结构，管理设计文件（.dsn）、库文件（.olb）、输出文件
  • 原理图编辑区：中间主区域，绘制原理图
  • 工具栏：顶部和右侧，最常用的是 Place 相关工具
  • Session Log：底部信息窗口，显示 DRC 结果、网表生成日志等

1.2 最关键的操作——放置元器件

OrCAD 中放置元器件的正确步骤：

  1. Place → Part（快捷键 P）— 弹出 Place Part 对话框
  2. 在 Libraries 中选中你要用的库。常用库路径：
  3. - C:\Cadence\SPB_17.4\tools\capture\library\*.olb（Cadence 自带库）
    - 自己创建的库 .\library\*.olb
  4. 搜索器件名，双击放置
  5. 按 R 旋转，按 Tab 放置多个
快捷键功能模拟工程师常用场景
PPlace Part（放置器件）最常用，放运放、电阻、电容
WPlace Wire（走线）连接引脚
NPlace Net Alias（网络别名）标注网络名，如 ADC_IN1
GPlace Ground（地）接地点，注意选 0/SOURCE

1.3 为模拟工程师建立的规范

规范 1：网络命名

好的网络命名能让原理图可读性大幅提升：

  模拟信号：ADC_IN1, ADC_IN2, DAC_OUT, TIA_OUT_CH1, PD_ANODE
  电源网络：+3V3_A（模拟 3.3V）, +3V3_D（数字 3.3V）, VCC_OPA, VEE_OPA
  参考电压：VREF_2V5, VREF_1V25
  地：GND_A（模拟地）, GND_D（数字地）, GND_PWR（电源地）

规范 2：添加电源和地符号

  • 每个 IC 必须有明确的电源和地连接——不要只依赖"隐含连接"，即使 OrCAD 支持 Off-page connector
  • 模拟地和数字地使用不同的符号，便于后续在 Allegro 中处理地平面分割
  • 使用 Place Power（快捷键 F）放置电源符号；使用 Place Ground（快捷键 G）放置地符号

规范 3：模拟信号添加测试点

  • 在所有关键模拟节点添加测试点（Test Point），包括：TIA 输出、ADC 输入、参考电压输出、电源输出
  • 在原理图中用特殊的符号标记测试点（Place → Test Point），或添加一个 0Ω 电阻占位
  • 这会在 Allegro 中自动生成测试点焊盘，方便调试时测量

规范 4：添加 DRC 标记

  • 模拟信号走线注意：Place → Line 画一条线，然后在线上右键 → Edit Properties，在 Net Properties 中添加约束
  • 例如：对 TIA 输入节点添加 NO_SLASH 属性（禁止自动灌铜），ROUTED 属性（必须手工走线）

1.4 创建设备符号（.olb）

如果 Cadence 标准库中没有你需要的器件，你需要自己创建：

  1. File → New → Library，创建 .olb 文件
  2. 在 .olb 上右键 → New Part
  3. 填写属性：
  4. - Name：器件型号，如 OPA381
    - Part Reference Prefix：位号前缀，如 U（IC）、R（电阻）、C（电容）
    - PCB Footprint：封装名，必须与 Allegro 中的封装名完全一致
    - Parts per Package：一个封装内的单元数，如 LM358 是双运放，选 2
  5. 用 Place Pin 放置引脚：
  6. - Power pins：VCC 选类型 Power，VEE 选类型 Power
    - Signal pins：输入选 Input、输出选 Output
    - 不用的引脚可以标注 NC
  7. 保存后即可在原理图 Place Part 中调用
创建运放符号的注意事项（模拟工程师必看）

1. 隐藏电源引脚：很多标准库中运放的 VCC/VEE 是隐藏引脚（hidden pin）。如果你在原理图中看不到电源连接，需要在 Part 属性中找到 Power Pins Visible 改为 True，或者通过 Place Power 符号连接。

2. 多个 Part 的位号：双运放（Parts per Package = 2）在原理图上显示为 U1A 和 U1B。PCB 上共用同一个封装。这是 Cadence 的 Heterogeneous 概念——原理图上的多 Part 对应 PCB 上的一个器件。

3. PCB Footprint 必须完全匹配：OrCAD 中的 PCB Footprint 属性值必须和 Allegro 中 .dra 文件的文件名一致（不含扩展名）。大小写不敏感但空格和特殊字符必须一致。最常见的网表错误就来自这里。

1.5 从 OrCAD 生成网表并导入 Allegro

  1. 原理图画完后，Tools → Create Netlist
  2. 在弹出的对话框中：
  3. - 选择 PCB Editor 标签页
    - PCB Footprint：勾上，确保检查所有器件有封装
    - Create PCB Editor Netlist：勾上
    - 点击 确定
  4. 成功生成后，会在项目目录下创建 netlistallegro 文件夹，包含 .dat、.net、.pst 文件
  5. 在 Allegro PCB Editor 中：File → Import → Logic（或直接 File → Import → Netlist）
  6. 选择 Cadence 标签，Import directory 指向 ./netlistallegro
  7. 点击 Import，完成网表导入

2. Allegro PCB Editor 布局布线

2.1 准备工作：叠层设计

在 Allegro 中开始布局之前，第一件事是配置 PCB 叠层：

  1. Setup → Cross-section（或 Setup → Stackup）
  2. 在 Cross-section Editor 中配置板层：

以 4 层模拟板为例：

层名类型厚度用途
TOPSignal1 oz (35 um)模拟信号走线 + 关键器件
GND_INNERPlane1 oz模拟地层（完整地平面）
PWR_INNERPlane1 oz电源层 + 必要时走少量信号
BOTTOMSignal1 oz次要信号 + 去耦电容

对于高精度模拟板，建议：

  • 顶层（TOP）放敏感模拟器件和走线
  • 第二层为完整的地平面（不分割）
  • 第三层为电源平面
  • 底层放不敏感的信号和辅助器件

2.3 模拟信号走线的 Allegro 技巧

技巧 1：建立模拟信号约束规则

  1. Setup → Constraints → Constraint Manager（或 Setup → Electrical Constraints）
  2. 在 Physical 标签页中，点击右键 → Create — Physical CSet
  3. 创建一个名为 ANALOG_SIGNAL 的物理约束集：
  4. - Line Width Min/Max：0.2 mm ~ 0.5 mm（根据电流需求调整）
    - Neck Width：0.15 mm（用于密集区域）
    - Vias：选择合适尺寸的过孔（如 0.3/0.5 mm 钻孔/焊盘）
  5. 在 Spacing 标签页中，创建 ANALOG_SPACE 间距约束集：
  6. - Line to Line：0.2 mm（模拟信号之间）
    - Line to Shape：0.3 mm（走线到铜皮）
    - Line to Via：0.2 mm
  7. 回到原理图或 Allegro 中，将模拟信号网络（如 TIA_OUT、ADC_IN）关联到这个约束集

技巧 2：保护环（Guard Ring）在 Allegro 中的实现

  1. 选择 Add → Line，在 Route Keepin 类中选择 Guard Ring（如果没有这个 subclass，可以自己创建）
  2. 在需要保护的高阻抗输入周围画一圈闭合走线
  3. 将这条线赋值为 GND 网络（选中后右键 → Assign Net → 选择 GND）
  4. 在 Shape → Add Rect 或 Add Circular 中选择 Guard Ring subclass，用铜皮覆盖这条走线
  5. 如果 Allegro 版本支持，也可以直接使用 Setup → Design Parameter Editor → Guard Ring 的自动化工具

技巧 3：模拟信号包地（Guard Trace）

  1. 对敏感模拟信号（如 ADC 的模拟输入），在其两侧各走一条 GND 线，起到屏蔽作用
  2. 操作：选 Add → Connect（快捷键 I），选择 GND 网络，在信号线两侧各走一条平行地线
  3. 在包地线上每隔一小段打一个地过孔

技巧 4：挖空输入节点下方的地铜（Keep-out）

  1. 选择 Shape → Void → Polygon 或 Shape → Void → Circular
  2. 在 TIA 输入焊盘下方画一个圆形或矩形区域
  3. 这会从地铜皮上挖掉这个区域，减少寄生电容

2.4 布局流程

  1. 放置器件（Place → Manually 或 Quickplace）：
  2. - 先放连接器（固定接口位置）
    - 再放核心 IC（ADC、运放、MCU）
    - 最后放外围无源器件（电阻电容就近摆放）
  3. 机械孔定位：先放螺丝孔，确认板卡外形
  4. 关键器件优先布局：
  5. - 光电二极管尽量靠近运放输入（< 3 mm）
    - Rf 和 Cf 紧靠运放
    - ADC 靠近模拟信号输入连接器
    - 去耦电容紧靠 IC 电源引脚
  6. 扇出（Fan-out）：对 QFP/TSSOP 封装的 IC，先将引脚引出到过孔
  7. 走线：先走模拟关键信号，再走普通信号，最后走电源
  8. 覆铜：先铺模拟地（GND_A），再铺数字地（GND_D），最后在背面用 0Ω 电阻或磁珠单点连接

2.5 Design Rule Check（DRC）

投板前必须做 DRC：

  • Setup → Constraints → Design Constraints — 检查和修改全局约束
  • Display → Status — 查看当前 DRC 状态。如果有错误图标，双击定位到错误位置
  • DRC 常见错误：
    • Line to Line Spacing：走线间距不足
    • Unrouted Net：有网络未连接
    • Unconnected Pin：有引脚没走线
    • Stub：走线分支过长（对高速信号需要关注，模拟信号可忽略）

2.6 出 Gerber 文件（投板前最后一步）

  1. Manufacture → Artwork（或 File → Export → Gerber）
  2. 在 Film Control 标签页中配置每层：
  3. - TOP：顶层走线 + 顶层阻焊（Solder Mask Top）
    - GND_INNER：内层地平面
    - PWR_INNER：内层电源层
    - BOTTOM：底层走线 + 底层阻焊
    - SILK_TOP：顶层丝印
    - DRILL：钻孔图
  4. 点击 Generate Artwork 生成 Gerber 文件
  5. 再生成钻孔文件：Manufacture → NC → Drill
  6. 用 CAM350 或 Allegro 自带的 Artwork Review 查看 Gerber 文件确认正确

3. OrCAD 和 Allegro 之间的交互

3.1 Cross-probe（交互定位）

这是 Cadence 流程中最常用的操作之一：

  • 在 OrCAD 中选中一个器件或网络 右键 → Select
  • 按下 Alt+Ctrl+F11（或点击工具栏的 Cross-probe 图标）
  • Allegro 会自动定位并高亮对应的器件或走线
  • 反过来在 Allegro 中也能高亮后 Cross-probe 到 OrCAD

3.2 Back Annotation（反向标注）

如果你在 Allegro 中修改了器件的位号（如交换了 U1 和 U2 的位号），需要回传到 OrCAD：

  1. Allegro 中：Logic → Back Annotation，生成 .swp 文件
  2. OrCAD 中：Tools → Back Annotate，导入 .swp 文件
  3. OrCAD 会自动更新原理图中的位号

4. 动手练习

练习 1：模拟 TIA 电路的 OrCAD 原理图操作

请写出在 OrCAD Capture 中完成以下操作的步骤：

  1. 放置一个 OPA381（SOIC-8）
  2. 放置一个光电二极管（使用标准库中的 PHOTODIODE 符号）
  3. 放置反馈电阻 Rf（Resistor）和反馈电容 Cf（Capacitor）
  4. 连接所有引脚（走线 + 标注网络名）
  5. 添加电源和地
  6. 生成网表
操作步骤：

Step 1：放置 OPA381
- 按 P 打开 Place Part
- 在 Libraries 中加载 OPA381 所在库（如果标准库没有，需要先创建 .olb）
- 双击 OPA381 放置到原理图
- 如果 OPA381 是双运放，会显示 U1A，可以再放一个 U1B

Step 2：放置光电二极管
- 按 P，在 DISCRETE.olb 中搜索 PHOTODIODE 或 LED
- 如果标准库没有光电二极管符号，可以自己创建一个 2-pin 的符号

Step 3：放置 Rf 和 Cf
- 按 P，在 DISCRETE.olb 选 R（电阻），C（电容）
- 在原理图上放置后，双击器件修改 Value（如 Rf = 100k）
- 在 Property Editor 中设置 PCB Footprint（如 R0603、C0402）

Step 4：走线
- 按 W 开始走线，点击光电二极管的阴极连接到 OPA381 的反相输入引脚（pin 2）
- Rf 一端连 pin 2，另一端连 pin 6（输出）
- Cf 并联在 Rf 两端
- 按 N 给输入节点命名 PD_IN，输出节点命名 TIA_OUT

Step 5：电源和地
- 按 F 放置电源 +5V 到 OPA381 的 VCC（pin 7）
- 按 F 放置电源 -5V 到 VEE（pin 4）
- 按 G 放置地到光电二极管的阳极

Step 6：生成网表
- Tools → Create Netlist
- 选 PCB Editor 标签页，确认
- 检查 Session Log 无错误

练习 2：Allegro 布局 TIA 电路的要点

在 Allegro 中布局一个 TIA 电路时，以下操作分别用什么命令/快捷键？

  1. 移动光电二极管使它与运放尽量靠近
  2. 手动走线连接 PD 到运放输入
  3. 在 TIA 输入焊盘下方挖掉地铜
  4. 创建模拟信号的线宽约束（0.3 mm）
  5. 高亮 TIA_OUT 网络检查走线路径
操作命令：

1. 移动器件：按 M（Move），点击光电二极管，移动到运放旁边。如果需要旋转，按 R。

2. 手动走线：按 I（Add Connect），点击 PD 焊盘，走到运放输入焊盘。如果需要换层，按 V 打过孔。

3. 挖空地铜：Shape → Void → Circular（或 Polygon），在输入焊盘下方画一个圈，从地铜皮上挖掉。

4. 约束设置：Setup → Constraints → Constraint Manager，在 Physical 标签页创建新的 Physical CSet，设 Line Width 为 0.3 mm。然后将 TIA 输入网络关联到此 CSet。

5. 高亮网络：按 Shift+F5，在 Options 面板中选择网络名 TIA_OUT，点击确定。

总结

OrCAD + Allegro 是业界最通用的 PCB 设计工具链之一。对模拟硬件工程师来说，关键不在于记住所有功能，而在于掌握从原理图到投板的完整流程：

OrCAD：画原理图 → 创建设备 → 分配封装 → DRC → 生成网表
Allegro：导入网表 → 设约束 → 布局 → 走线 → 覆铜 → DRC → 出 Gerber

多用快捷键（I、M、R、V），每次操作比点菜单快 3~5 秒，一天能省下大量时间。
