任务 3.1：创建自己的器件符号

标准库很好用，但实际工作中你会遇到标准库里没有的器件（比如你公司自研的芯片）。这时你需要自己创建原理图符号。

▶ 3.1.1 新建库文件

☐ 打开 OrCAD Capture
☐ File → New → Library
☐ 一个新的 .olb 文件出现在项目管理器中
☐ 立即保存：选中这个 .olb → File → Save As → 命名为 MY_PARTS.olb → 保存在你的工作目录下

▶ 3.1.2 创建一个 OPA381 符号

OPA381 是 TI 的一款低偏置电流运放，常用于 TIA 电路。
它有 8 个引脚（SOIC-8 封装）：
Pin #名称类型
1NC不接
2-IN反相输入
3+IN同相输入
4VEE负电源（这里接 GND）
5NC不接
6OUT输出
7VCC正电源
8NC不接
☐ 在 MY_PARTS.olb 上右键 → New Part
☐ 在弹出的 New Part Properties 窗口中填写：
  • Name：OPA381
  • Part Reference Prefix：U（表示这是一个 IC）
  • PCB Footprint：SOIC-8（这是 Allegro 中的封装名，大小写和拼写必须一致）
  • Parts per Package：1（单运放）
  • 点击 OK

▶ 3.1.3 放置引脚

引脚放置窗口会自动打开。如果关闭了，可以双击 .olb 下的 OPA381 重新进入编辑。
☐ 点击右侧工具栏的 Place Pin 图标（或按 Shift+P）
☐ 逐一放置 8 个引脚：
  • NC（不接）：Name = NC, Pin Number = 1, Shape = Line, Type = Passive
  • -IN（反相输入）：Name = -IN, Pin Number = 2, Shape = Line, Type = Input
  • +IN（同相输入）：Name = +IN, Pin Number = 3, Shape = Line, Type = Input
  • VEE（负电源）：Name = VEE, Pin Number = 4, Shape = Line, Type = Power
  • NC：Name = NC, Pin Number = 5, Shape = Line, Type = Passive
  • OUT（输出）：Name = OUT, Pin Number = 6, Shape = Line, Type = Output
  • VCC（正电源）：Name = VCC, Pin Number = 7, Shape = Line, Type = Power
  • NC：Name = NC, Pin Number = 8, Shape = Line, Type = Passive
💡 Type = Power 的引脚在原理图中可以选择"隐藏"（不显示）。对于 VCC/VEE，可以后面在原理图中用 F/G 连接，也可以设为隐藏并指定网络名。
☐ 按 Ctrl+S 保存库文件
☐ 关闭 Part Edit 窗口（点右上角的 X）

▶ 3.1.4 给符号画外形（可选，但建议）

☐ 重新双击 OPA381 进入编辑
☐ 用 Place Rectangle（放置矩形）画一个运放的三角符号
☐ 或直接不去管它——运放标不标注三角符号不影响电气连接

任务 3.2：用自建器件替换原理图

▶ 3.2.1 加载自建库

☐ 打开 Day 2 的原理图（或新建一个原理图页）
☐ 按 P → 点击 Add Library → 找到并加载 MY_PARTS.olb
☐ 在 Part List 中出现了 OPA381 → 双击放置

▶ 3.2.2 分配 PCB Footprint

如果你在创建符号时已经填了 PCB Footprint，可以跳过这一步。但如果之前没填，现在来补。
☐ 选中原理图中的 OPA381 → 按 Ctrl+E（Edit Properties）
☐ 在 Property Editor 中找到 PCB Footprint 列 → 改为 SOIC-8
☐ 对其他 Day 2 中的电阻电容也确认 PCB Footprint 已经填写（R0603、C0402 等）
⚠️ 这是最容易出错的地方！OrCAD 中的 PCB Footprint 名称必须和 Allegro 封装库中的文件名完全一致（大小写不敏感但拼写要一样）。

任务 3.3：Annotate + DRC

▶ 3.3.1 Annotate（自动整理位号）

Annotate 会自动把器件位号整理成 R1、R2、C1、C2、U1 这样的顺序。如果你不是从空白开始画，位号可能是乱的（比如 R1 和 R3 之间跳了 R2）。
☐ Tools → Annotate
☐ 在弹出的窗口中：
  • Action：选中 Incremental reference update（增量更新，只调整冲突的）
  • 也可以选 Reset part references to "?" → 再 Incremental（全部重置从 R1 开始）
  • 点击 OK
☐ 查看 Session Log 确认 "Annotate completed successfully"
☐ 检查原理图中位号是否已经是 R1、R2、C1、U1 这样的顺序

▶ 3.3.2 DRC（电气规则检查）

DRC 是原理图投板前必须做的检查。它会找出：浮空的输入引脚、单端网络（没有连东西的线）、短路、位号冲突等。
☐ Tools → Design Rules Check

☐ 在弹出的窗口中：
  • 勾选 Check single node nets（检查孤立网络——提高原理图质量的好习惯）
  • 不勾 Check off-page connector connections（除非有多页原理图）
  • 点击 OK
☐ 查看底部 Session Log：
  • 如果有 红色错误 → 双击错误行，OrCAD 会自动跳到错误位置
  • 常见错误及修复：
错误信息含义怎么修
Floating input pin输入引脚没连任何东西用 W 接上它，或者放一个 No Connect（X 键）标记它
Single node net XXX某网络只连了一个引脚检查这根线是不是忘了连接其他地方
Duplicate Reference有两个器件的位号相同重新跑一遍 Annotate
☐ 重复跑 DRC 直到 0 Errors（可以有 Warnings）
☐ 按 Ctrl+S 保存

任务 3.4：生成网表

▶ 3.4.1 生成网表

网表（Netlist）是连接 OrCAD 和 Allegro 的桥梁。OrCAD 把原理图的连接关系导出成网表文件，Allegro 读入后就知道哪些器件有哪些引脚、怎么连。
☐ Tools → Create Netlist
☐ 在弹出的窗口中：
  • 选择 PCB Editor 标签页
  • 勾选 Create PCB Editor Netlist
  • 勾选 PCB Footprint（检查所有器件都有封装名）
  • 点击 确定
☐ 看 Session Log：
  • ✅ Netlist successfully completed = 成功
  • ❌ Footprint not found = 有器件没设 PCB Footprint → 去补上

▶ 3.4.2 确认网表文件已生成

☐ 打开你的 Project 文件夹（在 Windows 资源管理器中）
☐ 确认有一个叫 netlistallegro 的文件夹
☐ 打开它，里面应该有：.dat、.net、.pst 等文件
💡 记下这个文件夹的路径（如 C:\Users\...\DAY3_PRACTICE\netlistallegro），Day 4 要用。

常见问题

问题原因解决方法
创建 .olb 后找不到没保存或路径不对File → Open → Library 可以打开 .olb 文件
PCB Footprint 不知道填什么不知道 Allegro 中封装的名称先随便填一个，在 Allegro 导入时会报错，根据报错修改
DRC 有几十个错误大部分是 NC 引脚没处理不用的引脚放 No Connect 符号（按 X 键，在引脚末端点击）
网表生成报"Footprint not found"有器件没填或填错了封装名按 Ctrl+E 检查所有器件的 PCB Footprint 属性
