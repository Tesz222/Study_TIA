SPI 协议 — 从时序图到固件实现

光电检测信号链 · Lesson 0011 ｜衔接：ADC (L05) · DAC8568 (L10)

关键洞察

SPI 看起来简单——就 4 根线，主设备发时钟，全双工同步传输。但实际工程中 >80% 的 SPI 故障出在同一个地方：主从设备的时钟极性/相位配置没配对。一节课搞懂 CPOL/CPHA，你就避开了 SPI 开发中最常见的坑。剩下的就是时序余量检查和 PCB 布局了。

1. SPI 是什么

SPI（Serial Peripheral Interface） 是由 Motorola 在 1980 年代提出的同步、全双工、主从式串行通信总线。它在嵌入式系统中是连接 MCU 与 ADC、DAC、传感器、存储器的头号选择。

传输方式同步（有专用时钟线）同步（时钟嵌入在数据中）异步

速度典型 10~50 MHz，可达 100+ MHz标准 100 kHz / 400 kHz / 1 MHz典型 115200 bps ~ 几 Mbps
寻址片选（硬件寻址）7/10 位地址（软件寻址）无

在信号链中ADC / DAC 的标准接口温湿度传感器、EEPROM调试终端、GPS 模块

Lesson 0005 的 ADS1256（Sigma-Delta ADC）和 Lesson 0010 的 DAC8568（精密 DAC）都使用 SPI 接口。在光测试仪表中，SPI 是你 MCU 和信号链芯片之间最主要的通信通道。

2. 四线信号

  ┌──────┐            ┌──────┐
  │ Master│            │Slave │
  │      │──── SCLK ──▶│      │  Serial Clock（时钟）
  │  MCU │──── MOSI ──▶│  ADC │  Master Out, Slave In
  │      │◀── MISO ────│  DAC │  Master In, Slave Out
  │      │──── CS  ──▶│      │  Chip Select（片选，低有效）
  └──────┘            └──────┘

  标准 4 线 SPI 连接（单从设备）

SPI 的核心概念：

  • SCLK（Serial Clock）：由主设备产生，决定数据传输速率。每个时钟周期传输 1 bit
  • MOSI（Master Out, Slave In）：主设备发送数据到从设备
  • MISO（Master In, Slave Out）：从设备发送数据到主设备。多设备共享此线，未选中设备必须三态（高阻）
  • CS（Chip Select，也称 SS、SYNC）：低电平时选中该从设备。每个从设备独占一根 CS 线
⚠ 术语注意

不同芯片厂商对 CS 的命名不同：TI 叫 CS 或 SYNC（DAC8568 中就是 SYNC），ADI 常用 CSB，NXP 用 SS。功能完全一样——低有效片选。
另外，MOSI/MISO 有时也叫 SDI/SDO（Serial Data In/Out）或 DI/DO。

3. SPI 的四个模式 — 核心中的核心

SPI 协议的精髓在于两个参数的排列组合——CPOL（时钟极性） 和 CPHA（时钟相位）。这两个位定义了数据在时钟的哪个边沿被采样和移出：

Mode 0 (CPOL=0, CPHA=0):
SCLK ──┐___┌─┐_┌─┐_┌─┐_┌─┐_┌─┐_┌─┐_┌─┐_┌──
 CS  ──┘                                 └──
MOSI ──┘  采样  采样  采样  采样  采样  采样
     空闲时 SCLK=低，上升沿采样（常用）

Mode 1 (CPOL=0, CPHA=1):
SCLK ──┐___┌─┐_┌─┐_┌─┐_┌─┐_┌─┐_┌─┐_┌─┐_┌──
 CS  ──┘                                 └──
     空闲时 SCLK=低，下降沿采样

Mode 2 (CPOL=1, CPHA=0):
SCLK ──┐‾‾‾└─┘‾└─┘‾└─┘‾└─┘‾└─┘‾└─┘‾└─┘‾└──
 CS  ──┘                                 └──
     空闲时 SCLK=高，下降沿采样

Mode 3 (CPOL=1, CPHA=1):
SCLK ──┐‾‾‾└─┘‾└─┘‾└─┘‾└─┘‾└─┘‾└─┘‾└─┘‾└──
 CS  ──┘                                 └──
     空闲时 SCLK=高，上升沿采样

一句话记住：

  • CPOL = 时钟空闲时的电平（0=低，1=高）
  • CPHA = 在哪个边沿采样（0=第一个边沿，1=第二个边沿）

3.1 常见器件默认模式

模式CPOLCPHA采样边沿移出边沿常见器件
Mode 000上升沿下降沿大多数 ADC/DAC（含 ADS1256、DAC8568）、Flash 存储器
Mode 101下降沿上升沿某些传感器（如部分温度传感器）
Mode 210下降沿上升沿某些 RF 收发器
Mode 311上升沿下降沿某些 EEPROM，NAND Flash
⚠ 最常见的 SPI 故障

MCU 的 SPI 外设默认初始化往往是 Mode 0。从设备数据手册中通常会给出时序图，图中明确标出"数据在 SCLK 上升沿被采样"——如果 MCU 配置成了下降沿采样，每个 bit 都错位采样，结果数据全是乱的。调试 SPI 的第一步：用示波器看 SCLK 和 MOSI，确认边沿对齐正确。

4. 时序参数

从设备数据手册中通常给出以下时序参数。读懂它们是成功通信的前提：

参数符号含义DAC8568 典型值
SCLK 频率fSCLK最高时钟速率50 MHz
SCLK 低电平时间tSCLKL时钟低电平最小宽度10 ns
SCLK 高电平时间tSCLKH时钟高电平最小宽度10 ns
CS 建立时间tCSSCS 拉低后，到第一个 SCLK 边沿的最小时间20 ns
CS 保持时间tCSH最后一个 SCLK 边沿后，CS 需保持低电平的最小时间10 ns

工程直觉：如果你的 MCU 运行在 80 MHz，SPI 外设最大时钟可设为 80/2 = 40 MHz。DAC8568 的最高 50 MHz 不是限制——但 PCB 走线长度会限制实际可用频率。经验法则：在 FR4 上，每 10 cm 走线引入约 0.5 ns 的传播延迟。40 MHz 时钟（周期 25 ns）下，走线长度超过 20 cm 就需要考虑信号完整性了。

5. 多设备连接

5.1 独立片选（最常用）

  ┌──────────┐
  │   MCU    │─── SCLK ────┬─── ADC (CS1)
  │          │─── MOSI ────┤
  │          │◀── MISO ────┤
  │          │─── GPIO1 ───┘
  │          │─── GPIO2 ─────── DAC (CS2)
  │          │─── GPIO3 ─────── 传感器 (CS3)
  └──────────┘

  每设备独占一根 CS → 可独立控制，互不干扰
  • 优点：每个设备独立控制，不限制数据帧长度，可以混用不同 SPI 模式的设备
  • 缺点：每增加一个设备多用一根 MCU GPIO
  • 在你的仪表中：ADC（ADS1256）独占一根 CS，DAC8568 独占另一根 CS——标准做法

5.2 菊花链

部分器件支持菊花链（Daisy-chain）连接：数据从 MOSI 流入第一颗，经过内部移位寄存器后从器件特定的 DOUT 引脚流入下一颗的 DIN，共用一根 CS。

  • 节省 GPIO，但要求所有器件支持菊花链模式
  • 数据帧长度 = 单帧长度 × 级联数量
  • DAC8568 不支持菊花链——但同系列的 DAC7568 支持

5.3 MISO 三态要求

所有从设备的 MISO 输出必须为漏极开路或三态。未选中的设备 MISO 必须进入高阻态，否则多个设备的输出会在 MISO 线上冲突——这是初学者最常犯的硬件错误。

6. 固件实现 — 以 DAC8568 为例

回顾 Lesson 0010 中的 DAC8568 SPI 编程，我们补充时序层面的细节：

// 1. SPI 外设初始化（以 STM32 HAL 为例） SPI_HandleTypeDef hspi1; hspi1.Instance = SPI1; hspi1.Init.Mode = SPI_MODE_MASTER; // 主模式 hspi1.Init.Direction = SPI_DIRECTION_2LINES; // 全双工（MOSI + MISO） hspi1.Init.DataSize = SPI_DATASIZE_8BIT; // 8 位一帧（注意：DAC8568 需要 24 位） hspi1.Init.CLKPolarity = SPI_POLARITY_LOW; // CPOL = 0 hspi1.Init.CLKPhase = SPI_PHASE_1EDGE; // CPHA = 0 → Mode 0 hspi1.Init.NSS = SPI_NSS_SOFT; // 软件控制 CS hspi1.Init.BaudRatePrescaler = SPI_BAUDRATEPRESCALER_8; // 80MHz/8 = 10MHz HAL_SPI_Init(&hspi1);
// 2. 发送 24 位指令帧到 DAC8568 // DAC8568 需要 24 位连续时钟，8 位 SPI 外设要发 3 次 uint8_t tx_buf[3]; uint32_t cmd = 0x800000 | (channel << 19) | dac_code; // 命令=10 (写并更新), 地址=channel, data=dac_code tx_buf[0] = (cmd >> 16) & 0xFF; // Bit 23-16: 命令+地址+保留 tx_buf[1] = (cmd >> 8) & 0xFF; // Bit 15-8: 数据高 8 位 tx_buf[2] = cmd & 0xFF; // Bit 7-0: 数据低 8 位 HAL_GPIO_WritePin(DAC_CS_GPIO_Port, DAC_CS_Pin, GPIO_PIN_RESET); // CS 拉低 HAL_SPI_Transmit(&hspi1, tx_buf, 3, 100); // 连续发送 3 字节（24 位） HAL_GPIO_WritePin(DAC_CS_GPIO_Port, DAC_CS_Pin, GPIO_PIN_SET); // CS 拉高
⚠ 24 位帧的陷阱

DAC8568 要求 24 位时钟连续、CS 在整个传输过程中保持低电平。如果你逐字节拉高 CS（很多初学者的做法），从设备会认为每一字节是一个独立的命令帧——数据会被解析成三个无效指令。务必在传输前拉低 CS、传输完 3 字节后再拉高 CS。

6.2 用示波器调试 SPI

当 SPI 通信不工作时，按以下顺序排查：

  1. 看 SCLK：有连续的时钟脉冲吗？频率对吗？
  2. 看 CS：传输期间持续低电平？有无中途抖动？
  3. 看 MOSI 数据边沿对齐：数据在 SCLK 采样边沿时是否稳定？（对照模式表确认）
  4. 看 MISO：从设备有响应吗？还是一直为低或高阻？
  5. 检查电气：电压电平是否兼容（3.3V MCU → 5V 器件需要电平转换？）

7. PCB 布局要点

复习 Lesson 0004 的 PCB 布局知识，结合 SPI：

信号布局要求理由
SCLK最短走线，少打过孔，远离模拟信号SCLK 是高速开关信号，容性耦合到模拟节点会引入噪声
MOSI / MISO与 SCLK 等长（高速时），避免 90° 拐角不等长导致数据与时钟的偏移（Skew）
CS加 10 kΩ 上拉电阻到 VDD防止 MCU 未初始化时 CS 浮空，误触发从设备
MISO如果不用，可悬空（从设备端设三态）半双工 SPI 时可以省掉 MISO 走线
SPI 总线与其他数字信号间隔 ≥ 3 倍线宽减少串扰
SPI 走线不要跨越分割的地平面返回路径不连续 → EMI 问题
🔗 与你已学的连接

Lesson 0004 中有一条规则："数字信号（如 SPI）与模拟输入区域之间至少保持 5 mm 间距"。这在混合信号 PCB 中尤其重要——10 MHz 的 SCLK 如果紧贴着 TIA 的反馈电阻走线，容性耦合进来的时钟噪声会直接出现在 TIA 输出上。

8. 练习与自测

问题 1：你在数据手册中看到一个 ADC 的时序图：CS 拉低后 SCLK 从低电平开始，数据在 SCLK 的第一个边沿（上升沿）被采样。这对应哪个 SPI 模式？

A. Mode 0 (CPOL=0, CPHA=0)
B. Mode 1 (CPOL=0, CPHA=1)
C. Mode 2 (CPOL=1, CPHA=0)
D. Mode 3 (CPOL=1, CPHA=1)
答案：A
SCLK 空闲低电平 → CPOL=0。在第一个边沿（上升沿）采样 → CPHA=0。这是最常用的 Mode 0，也是大多数 ADC 和 DAC（包括 ADS1256 和 DAC8568）的默认模式。如果你的 MCU 外设配置正确，第一件事就是确认 CPOL/CPHA 匹配 Mode 0。

问题 2：你的 PCB 上有两个 SPI 从设备：ADC（Mode 0）和温度传感器（Mode 3）。MCU 只有一组 SPI 外设。以下哪种方案正确？

A. 将两个设备都配置为 Mode 0，传感器会正常工作
B. 在每次片选切换前，重新配置 SPI 外设的 CPOL/CPHA
C. 用 GPIO 模拟 SPI 接口给温度传感器，硬件 SPI 给 ADC
D. 两个方案 B 和 C 都可以
答案：D
两种做法都可以。方案 B（重新配置外设）效率更高——在片选切换之间，重新写入 SPI 控制寄存器的 CPOL/CPHA 位，然后切换 CS。方案 C（GPIO 模拟）更灵活，适合低速传感器。实际工程中还有一个选择：换一个支持多组 SPI 外设的 MCU，将不同模式的设备分配到不同 SPI 总线。在光测试仪表中，通常 ADC 和 DAC 都在 Mode 0 下工作，很少遇到混合模式的问题。

问题 3：你调试 SPI 时发现 MISO 线上始终是低电平（0 V），但示波器确认 SCLK 和 CS 时序正确。以下哪种原因最可能？

A. 从设备的 MISO 引脚损坏
B. 从设备未被正确使能（CS 时序或电平不对）
C. SCLK 频率太高
D. MISO 线上有上拉电阻
答案：B
MISO 持续低电平最常见的原因是从设备没有使能，输出处于高阻态，而 MISO 线上的弱下拉（或没有上拉）让它读到 0。检查 CS 波形：它是否确实到了从设备的使能阈值？是否有噪声毛刺导致 CS 在传输中途短暂拉高？CS 是否在第一个 SCLK 边沿前已稳定拉低（满足 tCSS）？A 选项虽然可能，但概率远低于 B——芯片出厂损坏率极低。SCLK 频率太高通常不会导致 MISO 全低，而是数据乱码。

9. 小结

  • SPI = 同步 + 全双工 + 主从——ADC 和 DAC 的标准接口，在你的信号链中无处不在
  • 4 根线：SCLK（时钟）、MOSI（主发）、MISO（从发）、CS（片选）
  • 4 个模式由 CPOL（时钟空闲电平）和 CPHA（采样边沿）定义——>80% 的 SPI 故障源于此
  • Mode 0是默认模式——ADS1256（L05）和 DAC8568（L10）都在此模式下工作
  • 多设备用独立 CS 连接，MISO 三态输出，不能冲突
  • 调试顺序：看 SCLK → 看 CS → 看 MOSI 边沿对齐 → 看 MISO → 查电平兼容性
  • PCB 上：SCLK 最短、远离模拟信号、CS 加上拉、不走分割地平面
