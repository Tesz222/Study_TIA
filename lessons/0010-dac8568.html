DAC8568 — 八通道精密 DAC 与光测试仪表中的控制链路

光电检测信号链 · Lesson 0010 ｜数据手册：TI DAC8568

关键洞察

如果 ADC 是信号链的"眼睛"（读取测量结果），那么 DAC 就是信号链的"手"（执行控制动作）。DAC8568——16 位、8 通道、内置 2.5V 精密基准——正好对应你在 Lesson 0005（ADC）和 Lesson 0007（VOA）之间缺失的控制链路。它的一颗芯片可以同时控制 VOA 衰减、激光器偏置、光开关驱动——这正是多通道光测试仪表需要的。

1. 芯片速览

2.1 核心技术亮点

  • R-2R 梯形电阻网络：DAC8568 使用 R-2R 架构而非 PWM 或 Sigma-Delta 型，单调性有保证（16 位均单调），且无需输出滤波——这在你控制 VOA 时很重要，因为任何输出纹波都会直接调制到光功率上
  • 双缓冲结构（输入寄存器 → DAC 寄存器）：所有通道可以同时更新（LDAC 功能），确保多通道输出同步变化——适用于 WDM 通道同时均衡的场景
  • 内部基准 2.5V ±0.02%：节省一颗外部参考芯片（如 REF5050），但需要权衡——内部基准的温漂（5 ppm/°C typ）是否满足系统精度要求

3.1 VOA 驱动（核心场景）

回顾 Lesson 0007：MEMS VOA 需要 0~120 V 控制电压。DAC8568 输出 0~2.5 V（内部基准）或 0~5 V（外部基准），然后通过高压放大器（如 OPA454 或分立方案）升压到 VOA 所需的 0~120 V 范围。

DAC8568 在此处的关键优势：

  • 16 位分辨率：2.5 V 基准下 LSB = 38 µV，经升压放大到 120 V 范围后，等效分辨率 ≈ 0.0018 dB（VOA 输出端）
  • 单调性保证：确保衰减量随代码单调增加，不会出现"代码增大输出减小"的非单调问题
  • 8 通道：可同时控制多个 VOA（WDM 多通道）或控制 VOA + 激光器偏置 + 光开关

3.2 ADC 参考电压提供

虽然内部基准只有 2.5 V（而 Lesson 0005 的 ADS1256 可能需要 5 V 参考），但 DAC8568 可以接收外部基准。你可以用一个高精度参考（如 REF5050）提供给 DAC8568 的 VREF 引脚，DAC8568 内部的缓冲器同时为 AD 侧供电。

3.3 校准电压源

在仪表自校准流程中，DAC8568 的每个通道可以输出已知精确电压，用于：

  • 校准 ADC 的零点和满量程
  • 产生 TIA 的 DC 补偿电压（抵消运放失调）
  • 设置比较器的阈值

4. SPI 编程

DAC8568 使用标准的 4 线 SPI（SYNC、SCLK、DIN、LDAC），24 位指令帧格式：

  Bit 23-22   Bit 21-19   Bit 18-16   Bit 15-0
  ┌─────────┬──────────┬──────────┬─────────────────┐
  │ 命令     │ 地址      │ 保留      │ 数据 (16 位 DAC)  │
  │ (2位)    │ (3位)    │ (3位)    │                   │
  └─────────┴──────────┴──────────┴─────────────────┘

  命令编码:
    00 = 写输入寄存器 (只更新选中通道)
    01 = 更新 DAC 寄存器 (采用新数据)
    10 = 写输入 + 更新所有 DAC (写并同时更新)
    11 = 写控制寄存器 (配置内部基准、LDAC 等)

  地址编码:
    000 = 通道 A   100 = 通道 E
    001 = 通道 B   101 = 通道 F
    010 = 通道 C   110 = 通道 G
    011 = 通道 D   111 = 所有通道 (广播)
⚠ 常见陷阱

SPI 时序的常见错误：SYNC 必须在 24 个时钟周期内保持低电平。如果在传输中途拉高 SYNC，数据被丢弃。
此外，DAC8568 的 DIN 在 SCLK 下降沿采样——如果你的 MCU 在上升沿输出数据，需要配置 SPI 极性/相位（CPOL=0, CPHA=1）。

4.1 编程示例（伪代码）

// 初始化: 使能内部基准
SPI_Send(0b_11_111_000_0000000000000001);
// 命令=11 (写控制), 地址=111 (广播), data=0x0001 (bit0=1 使能内部基准)

// 设置通道 A 输出为 1.25 V (50% 满量程)
uint16_t code = (uint16_t)(1.25 / 2.5 * 65536);  // = 32768
SPI_Send(0b_10_000_000_0000000000000000 | code);
// 命令=10 (写并更新), 地址=000 (通道 A), data=code

// 同时更新所有通道到新值
// 先将各通道数据缓存到输入寄存器
SPI_Send(0b_00_000_000_0000000000000000 | chanA_code);  // 写通道 A 输入
SPI_Send(0b_00_001_000_0000000000000000 | chanB_code);  // 写通道 B 输入
// ...
// 然后拉低 LDAC 引脚 100 ns → 所有通道同时更新

5. 精度分析与基准配置

5.1 误差预算

在你的光测试仪表应用中，DAC8568 的总误差来源：

误差来源典型值对 VOA 控制的影响
INL±4 LSB = ±0.006% FSRVOA 衰减量的绝对精度误差。可在系统校准时补偿
内部基准初始精度±0.02% (2.5 V ± 0.5 mV)满量程偏差。同样可校准
内部基准温漂5 ppm/°C typ温度变化时的漂移。如果仪表在 25±5°C 环境下使用，贡献 25 ppm ≈ 0.0025%
增益误差±0.05% FSR typ输出放大器的增益偏差
失调误差±2 mV typ输出缓冲器零标偏移

工程直觉：如果你追求 0.1 dB 的 VOA 控制精度（这是测试仪表的典型要求），对应的电压精度约为 0.12%——DAC8568 的 INL（0.006%）远优于这个要求，但基准温漂可能在某些场景下成为限制因素。如果仪表需要在宽温度范围（如 0~50°C）下工作，考虑使用外部基准。

6. 使用注意事项

6.1 电源和基准去耦

将 Lesson 0008 的磁珠知识应用到这里：

  • VDD（引脚 16）：π 型滤波器（磁珠 100 Ω + 10 µF + 100 nF）
  • VREF（引脚 3）：100 nF 到 VREFGND，走线宽且短
  • VREFGND 与 GND 在 IC 下方单点连接——这两个地平面之间的噪声耦合会直接出现在 DAC 输出上

6.2 输出缓冲器稳定性

DAC8568 的内置输出缓冲器驱动容性负载时可能振荡。如果驱动远距离走线或大电容（>100 pF），建议串 50~100 Ω 隔离电阻（与 Lesson 0008 中 OPA2705 的处理相同）。

6.3 上电状态

DAC8568 上电时所有输出强制为 0 V（通过内部下拉电阻到 GND）。这在你设计 VOA 驱动时很重要：如果你希望上电时 VOA 衰减量为最大（安全状态），需要确保 DAC 输出 0 V 对应 VOA 的最大衰减配置。反之，如果 0 V 对应 0 dB，上电瞬间光功率会全通——在某些场景下可能损坏光接收机。

⚠ 安全设计关键

上电状态风险：如果 DAC8568 输出 0 V → VOA 驱动 0 V → VOA 衰减 0 dB（光全通），上电瞬间接收端可能收到满功率光信号。
解决方案：在 VOA 驱动电路设计中，确保驱动电压的默认状态对应最大衰减（而非最小）。可以在 DAC 输出端加分压偏置，或在高压级设置默认偏置。这点在 Lesson 0007 中已有提及，这里是具体实现。

7. 练习与自测

问题 1：DAC8568 使用内部基准（2.5 V）时，输出电压范围是 0~2.5 V。你需要 0~5 V 输出来驱动 VOA 控制电路。以下哪项正确？

A. 不能实现，DAC8568 的最大输出受 VDD 限制
B. 将 VDD 提高到 5.5 V，DAC8568 内部有 2× 增益放大器
C. 在 VREF 引脚输入 5 V 外部基准（如 REF5050），VDD 供电 5.5 V
D. 使用内部基准 + 后级 OPA2705 做 2× 放大
答案：C 和 D 都可实现，但 C 是更优方案
DAC8568 的 VREF 引脚可以接受外部基准电压（输入范围 0.5 V 到 VDD/2，以数据手册为准）。将 VREF 接到 5 V 外部基准，VDD = 5.5 V，输出范围即为 0~5 V——不需要额外放大级。方案 D（后级放大）虽然可行，但会引入额外噪声和失调。（注意：外部基准的电压范围有上限，具体值以数据手册为准。）

问题 2：你使用 DAC8568 的 8 个通道控制 8 个 VOA 通道。你需要所有通道同时切换到新衰减值。如何实现？

A. 依次向每个通道发送"写并更新"指令
B. 先向每个通道发送"写输入寄存器"指令，然后拉低 LDAC 引脚
C. 使用广播地址 0b111 一次更新所有通道
D. 将 CLR 引脚拉高再拉低
答案：B
DAC8568 的双缓冲结构正是为此设计的。步骤：1) 向每个通道发送"写输入寄存器"命令（命令=00），将新数据缓存到输入寄存器，输出维持不变。2) 所有 8 个通道的数据都缓存完毕后，将 LDAC 引脚拉低至少 100 ns。所有 DAC 寄存器同时从各自的输入寄存器加载新数据，输出同步变化。A 选项的"写并更新"会导致通道逐个更新，不同步。C 的广播地址会使所有通道输出相同值，不适合不同通道不同衰减的场景。

问题 3：DAC8568 的 INL = ±4 LSB（最大）。以下哪种说法正确？

A. 每个代码的电压误差不超过 ±4 LSB
B. DAC 的实际传输曲线与理想直线之间的最大偏差为 ±4 LSB
C. INL 导致输出噪声增加 4 LSB
D. INL 表示 DAC 的温度稳定度为 ±4 LSB
答案：B
INL（积分非线性）定义为实际传输曲线与端点拟合直线之间的最大偏差，单位 LSB。DAC8568 的 ±4 LSB 在 2.5 V 满量程下对应 ±153 µV。在 VOA 控制中，这个误差可以通过系统校准来补偿——在最终光功率输出端测量并记录偏差，然后反向修正 DAC 代码。这是一种典型的仪表校准流程。

8. 小结

  • DAC8568 = 16 位 × 8 通道 × 内置基准——光测试仪表控制链路的默认选择
  • 与 VOA（Lesson 0007）直接配套：DAC8568 输出控制电压 → 驱动 VOA → 改变光功率
  • 与 ADC（Lesson 0005）对称呼应：ADC 读测量，DAC 写控制，构成完整闭环
  • 内部基准够用：5 ppm/°C 温漂在常温仪表中已足够，宽温场景换外部基准
  • 双缓冲 + LDAC：多通道同步更新的工程实现
  • 安全注意：上电 0 V 输出可能对应 VOA 最小衰减，需在系统级设计中防范
